# 个人博客系统 PRD（产品需求文档）

---

## 1. 产品概述

### 1.1 项目名称

**NewBlog** — 极简个人博客系统

### 1.2 项目目标

构建一个轻量级、易部署的个人博客系统，让博主能够快速发布和管理文章内容，同时为读者提供清晰、专注的阅读体验。

### 1.3 核心价值

- **极简内容管理**：聚焦于写作和阅读，去除冗余功能，降低认知负担。
- **双端分离**：用户端专注浏览，管理后台专注内容维护，各司其职。
- **快速部署**：技术栈选型现代化，支持一键部署。
- **响应式优先**：一套代码同时适配桌面端和移动端，保证多设备阅读体验。

---

## 2. 目标用户

| 用户角色 | 描述 | 核心诉求 |
|---------|------|---------|
| **博主（管理员）** | 博客的拥有者，负责内容创作和系统维护 | 便捷地撰写、编辑和管理文章；查看博客整体数据概况 |
| **普通读者** | 通过链接访问博客的访客 | 流畅地浏览文章列表、阅读完整文章内容、了解博主背景 |

## 3. 功能需求

### 3.1 用户端

#### 3.1.1 首页

| 项目 | 说明 |
|------|------|
| **功能描述** | 作为博客的门面，展示最新发布的文章列表，吸引读者点击阅读 |
| **核心交互** | 文章列表展示、分页加载、点击进入详情 |
| **页面组件** | |
| | **导航栏 (Navbar)**：站点名称/Logo、导航菜单（首页、关于我）、响应式折叠菜单（移动端汉堡菜单） |
| | **文章列表区 (ArticleList)**：文章卡片（标题、摘要、发布日期、分类），每页默认显示 10 篇 |
| | **分页器 (Pagination)**：上一页/下一页按钮、当前页码指示 |
| | **页脚 (Footer)**：版权信息、社交链接（可选）、备案号 |
| **数据来源** | 从后端 API 获取已发布文章列表，按发布时间倒序排列 |

#### 3.1.2 文章详情页

| 项目 | 说明 |
|------|------|
| **功能描述** | 展示单篇文章的完整内容，提供良好的长文阅读体验 |
| **核心交互** | Markdown 内容渲染、回到顶部、相关文章推荐（可选） |
| **页面组件** | |
| | **文章头部 (ArticleHeader)**：文章标题、作者、发布日期、分类 |
| | **文章正文区 (ArticleBody)**：Markdown 渲染的富文本内容，支持代码高亮、图片展示、表格等 |
| | **文章底部 (ArticleFooter)**：上一篇/下一篇导航链接 |
| | **评论区占位 (CommentPlaceholder)**：（可选）第三方评论系统集成（如 Giscus、Waline） |
| **数据来源** | 从后端 API 根据文章 slug/id 获取单篇文章内容 |

#### 3.1.3 关于我页面

| 项目 | 说明 |
|------|------|
| **功能描述** | 博主个人介绍页面，让读者了解博客背后的作者 |
| **核心交互** | 静态内容展示 |
| **页面组件** | |
| | **个人信息区 (ProfileSection)**：头像、昵称、一句话简介 |
| | **详细介绍区 (BioSection)**：Markdown 渲染的自述内容（个人经历、技能、兴趣等） |
| | **社交链接区 (SocialLinks)**：GitHub、邮箱、Twitter/X 等链接图标 |
| **数据来源** | 从后端 API 获取博主个人信息数据 |

---

### 3.2 博主管理后台

#### 3.2.1 管理员登录

| 项目 | 说明 |
|------|------|
| **功能描述** | 博主的专属登录入口，通过账号密码验证身份，获取后台管理权限 |
| **核心交互** | 表单输入、登录验证、Token 持久化、登出 |
| **页面组件** | |
| | **登录表单 (LoginForm)**：账号输入框、密码输入框、登录按钮、错误提示信息 |
| | **登录背景 (LoginBackground)**：简洁的居中登录卡片布局 |
| **业务规则** | 账号密码在后端验证，成功后返回 JWT Token 或 Session Cookie；连续失败 5 次锁定 15 分钟；Token 过期后自动跳转登录页 |

#### 3.2.2 后台仪表盘

| 项目 | 说明 |
|------|------|
| **功能描述** | 登录后的默认页面，以数据可视化的方式展示博客整体状况 |
| **核心交互** | 数据卡片浏览、快速跳转 |
| **页面组件** | |
| | **侧边栏导航 (Sidebar)**：仪表盘、文章管理、返回前台、退出登录 |
| | **顶部栏 (TopBar)**：当前登录用户名、退出按钮 |
| | **统计卡片区 (StatsCards)**：文章总数、已发布数、草稿数、分类数 |
| | **快捷入口 (QuickActions)**：新建文章按钮、跳转文章管理 |
| **数据来源** | 从后端 API 获取聚合统计数据 |

#### 3.2.3 文章管理

| 项目 | 说明 |
|------|------|
| **功能描述** | 文章的核心管理模块，支持列表查看、新建、编辑、删除操作 |
| **核心交互** | 表格浏览、表单编辑、确认删除、搜索过滤 |
| **页面组件** | |
| | **文章列表表格 (ArticleTable)**：列（标题、状态、分类、发布时间、操作）；状态筛选（全部/已发布/草稿）；搜索框（按标题模糊搜索） |
| | **新建/编辑文章页 (ArticleEditor)**：标题输入框、分类选择器、Markdown 编辑器（支持实时预览）、发布状态切换（草稿/发布）、保存按钮、取消按钮 |
| | **删除确认弹窗 (DeleteConfirmModal)**：二次确认提示，防止误删 |
| **业务规则** | 删除为软删除（回收站逻辑），草稿仅博主可见，已发布文章在用户端公开可见 |

## 4. 非功能性需求

### 4.1 响应式设计

| 断点 | 对应设备 | 布局策略 |
|------|---------|---------|
| **≥ 1024px** | 桌面端 | 用户端：文章列表最大宽度 800px 居中、文章详情最大宽度 720px 居中；管理后台：侧边栏固定左侧、内容区自适应 |
| **768px ~ 1023px** | 平板端 | 用户端：内容宽度自适应；管理后台：侧边栏可折叠 |
| **< 768px** | 手机端 | 用户端：导航栏变为汉堡菜单、文章卡片占满屏宽；管理后台：侧边栏隐藏，通过顶部汉堡按钮呼出、表格转为卡片布局 |

### 4.2 性能要求

- 首屏加载时间（LCP）< 2.5 秒
- 交互响应时间（FID）< 100 毫秒
- 页面切换过渡流畅，无白屏闪烁

### 4.3 安全要求

- 管理后台所有接口需携带有效 Token 验证
- 密码存储使用 bcrypt 等加盐哈希算法
- API 接口做基础的速率限制（防暴力破解）

### 4.4 兼容性

- 支持 Chrome、Firefox、Safari、Edge 最近 2 个大版本
- 移动端支持 iOS Safari、Android Chrome 最近 2 个大版本
